**Why:** 요약 문서와 원본 공식 docs 간 사실 관계 검증 요청

**How to apply:** 향후 이 파일 재검토 시 아래 오류 이력 참고

## 발견된 오류/누락 패턴 (2026-04-01 기준)

1. **자동 메모리 목적 설명 누락**: 기본 개념 도입 문장이 자동 메모리의 핵심 특성(Claude가 스스로 학습 축적)을 충분히 설명하지 않음
2. **우선순위 방향 혼란**: 사용자 수준 규칙 섹션에서 "먼저 로드되어 프로젝트 규칙이 높은 우선순위를 가진다"는 인과관계가 불명확. 원본: "프로젝트 규칙에 더 높은 우선순위를 제공합니다" — 나중에 로드될수록 높은 우선순위를 갖는 메커니즘
3. **autoMemoryDirectory 보안 이유 누락**: 프로젝트 설정 제외 이유("공유 프로젝트가 민감한 위치로 메모리를 리디렉션하는 것을 방지")가 누락. 허용 범위(정책/로컬/사용자) 자체는 정확
4. **MEMORY.md 200줄 제한 범위 미구분**: 200줄 제한이 MEMORY.md에만 적용되고 CLAUDE.md는 전체 로드된다는 구분이 어디에도 없음. 독자가 CLAUDE.md도 200줄만 로드된다고 오해할 수 있음
5. **`--add-dir` 기본 동작 누락**: 환경 변수 없이 `--add-dir`만 사용하면 추가 디렉토리의 CLAUDE.md가 기본적으로 로드되지 않는다는 중요한 맥락 누락
6. **/memory 토글 표현 오해 소지**: `/memory` 명령이 토글 명령처럼 기술되어 있으나, 실제로는 UI를 열고 그 내부에서 토글 사용

## 정확하게 기술된 항목
- 모든 경로(관리 정책/프로젝트/사용자 수준), 환경 변수, 설정 키 이름, 명령어, 버전 요구사항(v2.1.59), @path 5홉 깊이 제한, glob 패턴 표, claudeMdExcludes 예시 모두 정확
- 자동 메모리 저장 경로, git 기반 프로젝트 경로 파생, /compact 후 재로드, 심볼릭 링크 예시 모두 정확
