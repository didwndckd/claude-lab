# 컨텍스트 윈도우

> https://code.claude.com/docs/en/context-window

## 기본 개념

- 컨텍스트 윈도우는 Claude Code가 한 세션에서 사용할 수 있는 토큰의 총량이다.
- 세션이 시작되면 시스템 프롬프트, 메모리, 환경 정보 등이 자동으로 로드되어 컨텍스트를 채운다.
- 대화가 진행될수록 파일 읽기, 도구 출력, 사용자 메시지 등이 누적된다.
- 컨텍스트가 한계에 가까워지면 자동 압축(compaction)이 수행된다.

### 가시성 수준

| 수준     | 설명                                      |
| ------ | --------------------------------------- |
| hidden | 터미널에 표시되지 않는다. 사용자는 볼 수 없지만 컨텍스트를 차지한다. |
| brief  | 터미널에 한 줄 요약만 표시된다. 실제 내용은 Claude만 본다.   |
| full   | 터미널에 전체 내용이 표시된다.                       |

## 컨텍스트 로딩 순서

### 1단계: 시작 시 자동 로드

| 항목                      | 토큰(예시) | 가시성    | 설명                                                       |
| ----------------------- | ------ | ------ | -------------------------------------------------------- |
| System prompt           | ~4,200 | hidden | 행동, 도구 사용, 응답 포맷 관련 핵심 지시사항                              |
| Auto memory (MEMORY.md) | ~680   | hidden | 이전 세션에서 Claude가 기록한 메모. 처음 200줄 또는 25KB 중 먼저 도달하는 것까지 로드 |
| Environment info        | ~280   | hidden | 작업 디렉토리, 플랫폼, 셸, OS 버전, git 상태                           |
| MCP tools (deferred)    | ~120   | hidden | MCP 도구 이름 목록. 전체 스키마는 필요 시 로드                            |
| Skill descriptions      | ~450   | hidden | 사용 가능한 스킬 한 줄 설명. `/compact` 후 재주입되지 않음                  |
| ~/.claude/CLAUDE.md     | ~320   | hidden | 전역 사용자 설정                                                |
| Project CLAUDE.md       | ~1,800 | hidden | 프로젝트 규칙, 빌드 명령, 아키텍처 노트                                  |

### 2단계: 대화 중 로드

| 항목           | 가시성    | 설명                                                                             |
| ------------ | ------ | ------------------------------------------------------------------------------ |
| 사용자 프롬프트     | full   | 사용자가 입력한 메시지                                                                   |
| 파일 읽기        | brief  | `Read` 도구로 읽은 파일. 컨텍스트 사용의 대부분을 차지                                             |
| 경로 기반 규칙     | brief  | `.claude/rules/`에 `paths:` 패턴이 있는 규칙. 매칭 파일 접근 시 자동 로드                         |
| 검색 결과        | brief  | `grep`, `glob` 등의 검색 출력                                                        |
| Claude 분석/편집 | full   | Claude의 분석, 설명, 코드 수정                                                          |
| Hook 출력      | hidden | 모든 훅 타입에서 `hookSpecificOutput.additionalContext` 필드로 컨텍스트에 진입. 최대 10,000자      |
| 테스트/명령 출력    | brief  | `npm test` 등 명령 실행 결과                                                          |
| `!` 명령       | full   | 사용자가 `!` 접두사로 실행한 셸 명령과 출력                                                     |
| 스킬           | brief  | 슬래시 명령으로 호출 시 전체 내용 로드. `disable-model-invocation: true`인 스킬은 호출 전까지 컨텍스트 비용 0 |

### 3단계: 서브에이전트

서브에이전트는 **별도의 컨텍스트 윈도우**를 사용한다. 메인 세션의 컨텍스트를 소비하지 않는다.

| 항목 | 설명 |
|------|------|
| System prompt | 메인보다 짧은 자체 시스템 프롬프트. 메인 세션의 auto memory 미포함 |
| CLAUDE.md | 같은 파일이지만 서브에이전트의 컨텍스트에서 카운트 |
| MCP/Skills | 부모와 동일한 MCP 서버·스킬 접근 가능. 일부 도구(플랜 모드, Agent 등) 제외 |
| Task prompt | 사용자 프롬프트 대신 Claude가 작성한 작업 지시를 받음 |
| 파일 읽기 | 서브에이전트의 컨텍스트만 사용. 메인에 영향 없음 |
| 반환 | 최종 텍스트 응답 + 메타데이터만 메인 컨텍스트에 추가 |

### 4단계: 컴팩션 (`/compact`)

- 전체 대화를 **구조화된 요약**으로 대체한다.
- 원본 토큰의 약 **12%** 크기로 압축된다.
- 보존 항목: 사용자 요청·의도, 핵심 기술 개념, 파일 목록·코드 스니펫, 에러·수정 내역, 대기 작업
- 제거 항목: 전체 도구 출력, 중간 추론 과정
- 시작 시 자동 로드 항목(시스템 프롬프트, 메모리, CLAUDE.md 등)은 컴팩션 후에도 유지된다.
- 단, 스킬 설명 목록은 재주입되지 않는다. 실제 호출한 스킬만 보존된다.

## 옵션

| 옵션                                    | 설명                                                              | 기본값     |
| ------------------------------------- | --------------------------------------------------------------- | ------- |
| `ENABLE_TOOL_SEARCH` 미설정              | 모든 MCP 도구 스키마를 온디맨드로 지연 로드                                      | 기본 동작   |
| `ENABLE_TOOL_SEARCH=true`             | 미설정과 동일. 모든 MCP 도구 스키마를 온디맨드로 지연 로드                             | -       |
| `ENABLE_TOOL_SEARCH=auto`             | 전체 MCP 도구 스키마 크기가 컨텍스트 윈도우의 10% 이하이면 전부 즉시 로드, 초과하면 온디맨드로 지연 로드 | -       |
| `ENABLE_TOOL_SEARCH=auto:<N>`         | `auto`와 동일하되, 임계값을 N%로 지정                                       | -       |
| `ENABLE_TOOL_SEARCH=false`            | 모든 MCP 도구 스키마를 처음부터 전부 로드                                       | -       |
| `disable-model-invocation: true` (스킬) | 스킬을 모델 호출 목록에서 제외. 사용자가 `/name`으로 호출할 때만 로드                     | `false` |

## 사용 예시

### 컨텍스트 절약: 서브에이전트 활용

```
사용자: "Use a subagent to research session timeout handling, then fix it"
```

- 서브에이전트가 6,100 토큰의 파일을 읽고 420 토큰의 요약만 메인에 반환한다.
- 리서치 작업을 서브에이전트에 위임하면 메인 컨텍스트를 크게 절약할 수 있다.

### 컨텍스트 절약: 구체적 프롬프트 작성

```
# 비효율적 — Claude가 많은 파일을 탐색
"auth 관련 버그 수정해줘"

# 효율적 — Claude가 필요한 파일만 읽음
"auth.ts에서 토큰 갱신 후 401이 발생하는 버그 수정해줘"
```

### 컨텍스트 절약: 스킬에 disable-model-invocation 설정

```yaml
# 스킬 프론트매터
---
disable-model-invocation: true
---
```

- 커밋, 배포, 메시지 전송 등 부작용이 있는 스킬에 설정한다.
- 사용자가 `/name`으로 호출할 때까지 컨텍스트 비용이 0이다.

### 컴팩션 실행

```
사용자: /compact
```

- 대화가 길어져 컨텍스트가 부족할 때 수동으로 실행한다.
- 컨텍스트 한계에 근접하면 자동으로도 수행된다.
